About the fields
Account

The account the cash/cheque or debit is being paid out of. You canâ€™t amend this.

Ledger

The Ledger (if required) code you want to assign the transaction to. You can select the code from the drop-down list of available codes.

Ledger codes are set up within Administration. These codes allow you to continue providing unified reports (e.g. VAT reporting) when you integrate with external financial systems. Setting up and assigning the ledger code which matches the ledger code in your external system ensures that your transactions will fall in the correct ledger for reporting.

Description

A description for the payment. This should be something which makes it easy to identify, for example Trip to Isle of Man, Unreconciled transaction receipt from previous system, etc.

Type

The type of payment. This can be Cash, Cheque, Direct DebitPayment direct out of a person's or organisation's bank account. For example, payment via BACS or bill payment. or ePaymentPayment made across the Internet, for example, parent online transactions. (payment). You canâ€™t amend this.

Gift Aid

Indicates whether the receipt is eligible for gift aidThis is a UK government scheme which lets registered charities reclaim a percentage of tax on donations from UK taxpayers.. Selected means that it is eligible. Clear indicates it is not.

Note: To be eligible, your organisation must be a charity, or have charitable status and the person making the payment must be a UK taxpayer. For more information, see HMRC requirements for claiming gift aid.

Notes

Notes about the payment.

VAT Code

The VAT rate for the transaction. You can select from a number of pre-defined VAT rates as follows: Zero (0%), Standard (20%), Reduced (5%), Exempt (0%), Outside scope (0%), or Custom (which you define yourself).

For guidance on VAT rates, see the HMRC guidance - VAT rates on different goods and services.

VAT Rate

Only applicable if you select Custom for the VAT Code. Enter the % VAT rate to apply.

Reference

A reference you enter which helps you to identify or trace the payment. You canâ€™t amend this.

Date

The date the transaction takes place for VAT purposes.

The transaction date defaults to todayâ€™s date but you can amend this, if required. For example, you can enter the date of payment as the date on a cheque, or the date of the direct debit.

Amount

The gross amount of the transaction or the individual Amount to bulk input. You canâ€™t amend this.

Chq/Vouch No

The number which is automatically generated within Fund Manager. You can change this number when you create the payment and you can change it later if you amend the payment.

Name

The pupil name, class, year, registration group, groups, and status (whether the pupil name is in or out of use).

The Amend Payment screen displays the information for the transaction you selected on the Amend Receipt/Payment screen.

  • You can amend the Ledger, Description, VAT Code, VAT Rate, Date and Receipt No.
  • Click Save when you have finished amending the details you want to amend.
